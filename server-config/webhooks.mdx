<Note>
    This requires process the `manage_webhooks` permission.
</Note>

## Create a webhook
Command:`;webhook create` <br />
Description: Create a new webhook. <br />

<Tabs>
    <Tab title="Example">
        <Frame>
            <img src="https://media.discordapp.net/attachments/1334725117221015582/1334725313548124240/image.png?ex=679d9350&is=679c41d0&hm=9f9392cc63ef0f6470feefbe72e4052b1fb6c7eb5a02bf230ebfaae75bf1e424&=&format=webp&quality=lossless"/>
        </Frame>
    </Tab>
    <Tab title="Syntax">
       ```
       ;webhook create [channel] [name]
       Example: ;webhook create #general rules!
       ```
    </Tab>
</Tabs>

<Tip>
    You may want to copy the indentifier if you're going to utilize the webhook right afterwards. If lost, use `;webhook list` to retrieve it.
</Tip>



## Sending messages with webhooks
Command:`;webhook forward` <br />
Description: Forward a message through a webhook. <br />

<Tabs>
    <Tab title="Example">
        <Frame>
           <img src="https://media.discordapp.net/attachments/1334725117221015582/1334732494704218265/image.png?ex=679d9a00&is=679c4880&hm=d44f0aaee7f21eba2fc2d21a7e5d6344400de03ef00ee9152d1ae478859f683a&=&format=webp&quality=lossless"/>
        </Frame>
    </Tab>
    <Tab title="Syntax">
       ```
       ;webhook forward (identifier) (script or message)
       Example: ;webhook forward 6JwFAGlQ Hello, how are you?
       ```
    </Tab>
</Tabs>



## Editing webhook messages
Command:`;webhook edit` <br />
Description: Edit a message sent by a webhook. <br />

    </Tab>
    <Tab title="Output">
        <Frame>
           <img src="https://media.discordapp.net/attachments/1334725117221015582/1334740617141555210/image.png?ex=679da191&is=679c5011&hm=48ef3344ea197f077bacfb4dbf51c8845d7533cca589f721143ac0a197c44734&=&format=webp&quality=lossless"/>
        </Frame>
    </Tab>
    <Tab title="Syntax">
    ```
    Syntax: ;webhook edit (message id) (script or message)
    Example: ;webhook forward 6JwFAGlQ Hello, how are you? 
    ```
    </Tab>
</Tabs>
<Warning>
    This doesn't customize the name, profile, or how the webhook looks whatsoever, rather a message it sent.
</Warning>
